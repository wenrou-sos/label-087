## 1. 产品概述

消防救援队伍作战指挥平台是一套面向消防指挥中心的实战化指挥调度系统，旨在实现从接警录入、智能调度、地图指挥、实时监测到事后归档的全流程数字化管理。目标用户为消防指挥中心调度员、现场指挥员和后勤管理人员。

- 解决传统接警调度流程中信息传递慢、调度决策依赖经验、现场态势感知不足等核心痛点
- 通过GIS空间分析、实时数据推送和可视化指挥，实现"接警-调度-指挥-处置-归档"全链路数字化闭环

## 2. 核心功能

### 2.1 用户角色

| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 指挥调度员 | 管理员分配账号 | 接警录入、智能调度、地图指挥、实时监测 |
| 现场指挥员 | 管理员分配账号 | 查看调度方案、地图指挥、实时数据查看 |
| 系统管理员 | 系统内置 | 用户管理、系统配置、日志审计 |
| 数据分析员 | 管理员分配账号 | 事后数据管理、统计分析、报告生成 |

### 2.2 功能模块

1. **指挥总览大屏**：全局态势感知、实时警情统计、资源概览、快捷入口
2. **警情信息管理**：接警录入、警情编辑、历史查询、警情状态追踪
3. **智能调度系统**：距离计算、力量匹配、出动方案生成、车辆状态跟踪
4. **数字化地图指挥**：火点标注、水源标注、路线规划、警戒范围划定
5. **实时数据监测**：温度热力图、气体浓度监测、人员定位追踪、阈值告警
6. **事后数据管理**：损失录入、原因调查、处置总结、数据归档

## 3. 核心流程

### 3.1 接警调度主流程

指挥中心接警后，调度员在系统中快速录入警情信息，系统基于GIS空间分析自动推荐最近的消防站和最优救援力量组合，调度员确认方案后下达出动指令，现场指挥员通过地图指挥模块实时掌控态势，单兵设备数据回传至监测大屏，处置完成后进入事后归档。

```mermaid
flowchart TD
    "接警录入" --> "信息校验"
    "信息校验" --> "GIS空间分析"
    "GIS空间分析" --> "智能推荐方案"
    "智能推荐方案" --> "调度员确认"
    "调度员确认" --> "下达出动指令"
    "下达出动指令" --> "车辆出动"
    "车辆出动" --> "地图实时指挥"
    "地图实时指挥" --> "现场数据回传"
    "现场数据回传" --> "实时监测告警"
    "实时监测告警" --> "处置完成"
    "处置完成" --> "事后数据归档"
```

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：深蓝黑(#0A1628)为背景底色，搭配消防红(#E53E3E)作为警示强调色，天际蓝(#3B82F6)作为信息交互色
- **辅助色**：琥珀色(#F59E0B)用于告警状态，翡翠绿(#10B981)用于安全/正常状态
- **按钮风格**：圆角矩形(8px)，主操作按钮使用实心填充，次要操作使用描边样式
- **字体**：数字/数据使用等宽字体 JetBrains Mono，中文正文使用思源黑体/Noto Sans SC
- **布局风格**：左侧固定导航栏 + 顶部状态栏 + 主内容区，指挥大屏采用全屏沉浸式布局
- **图标风格**：线性图标，2px描边，搭配微光效果用于关键状态指示

### 4.3 响应式设计

- **指挥大屏(≥1920px)**：全屏沉浸式布局，多面板并排，大字体数据展示
- **PC端(1024px-1919px)**：侧边导航+主内容区，适度缩放
- **平板(768px-1023px)**：底部导航，卡片堆叠布局
- **手机(< 768px)**：底部Tab导航，全屏卡片式切换，触摸优化

### 4.4 动效设计

- 页面切换：淡入淡出 + 轻微位移
- 数据更新：数字滚动计数器动画
- 新警情：红色脉冲边框动画
- 车辆移动：地图上平滑轨迹移动
- 告警触发：顶部红色横幅滑入 + 声音提示
- 图层切换：透明度渐变过渡
